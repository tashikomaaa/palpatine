Usage

Interactive Mode

Launch without arguments for the main menu:

./palpatine

Navigate with keyboard to scan systems, execute commands, reboot fleet, manage plugins, and more.

CLI Mode

Scan Fleet Status

./palpatine --group prod --action status

Execute Command Across Fleet

./palpatine --action run --cmd "df -h"
./palpatine --action run --cmd "systemctl restart nginx"

Reboot with Dry-Run

./palpatine --action reboot --dry-run

Focus on Server

./palpatine --focus "root@web-01"
./palpatine --focus 2  # Server number from list

Run Playbook

./palpatine --playbook playbooks/nightly-backup.playbook

Playbook Directives

Directive Description
scan Run fleet scan workflow
run: <command> Execute command across all servers
reboot Reboot all servers (no confirmation)
shutdown Shutdown all servers (no confirmation)
sleep: <seconds> Wait between steps
note: <message> Log message to output

Configuration

Palpatine reads config files in this order (later overrides earlier):

  1. /etc/palpatine.conf
  2. ~/.palpatine.conf
  3. ./.palpatine.conf

Example Configuration

# ~/.palpatine.conf
GROUP="production"
SSH_USER="deploy"
MAX_JOBS=16
SSH_TIMEOUT=10
UI_LANG="en"
DRY_RUN=false

# Scan output
SCAN_OUTPUT_JSON=true
SCAN_OUTPUT_DIR="$HOME/palpatine_scans"
SCAN_REPORT=true
SCAN_INTERACTIVE_RETRY=true

# Auto-update
DISABLE_UPDATE_CHECK=false
UPDATE_CHECK_INTERVAL=86400
UPDATE_REMOTE=origin
UPDATE_BRANCH=main

Configuration Variables

Variable Description Default
GROUP Default server group (uses servers-GROUP.txt) -
SSH_USER Default SSH username for bare hostnames root
MAX_JOBS Maximum parallel SSH jobs 10
SSH_TIMEOUT SSH ConnectTimeout in seconds 10
UI_LANG Interface language (en, fr, de, es, pt, it, ru, uk, pl, ja, ko, zh) en
DRY_RUN Simulate actions without execution false
SCAN_OUTPUT_JSON Enable JSON scan output false
SCAN_REPORT Enable Markdown scan reports false
